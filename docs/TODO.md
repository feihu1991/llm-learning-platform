## 📝 开发建议

### 1. 开发顺序
1. 先完成后端基础架构
2. 实现核心业务逻辑
3. 开发前端界面
4. 进行集成测试
5. 优化和部署

### 2. 代码规范
- 遵循Java编码规范
- 遵循Vue组件规范
- 使用ESLint和Prettier
- 编写单元测试

### 3. 提交规范
- feat: 新功能
- fix: 修复bug
- docs: 文档更新
- style: 代码格式
- refactor: 重构
- test: 测试相关
- chore: 构建过程或辅助工具的变动

### 4. 分支管理
- main: 主分支（生产环境）
- develop: 开发分支
- feature/xxx: 功能分支
- hotfix/xxx: 热修复分支
